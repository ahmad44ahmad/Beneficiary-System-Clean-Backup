### 2.2 الضوابط المُعتمَدة على الوكالة عند التسليم

هذه الضوابط **لا يُنفّذها فريق التطوير الحالي** لأنّها تتطلّب قرارات أو موارد من مستوى الوكالة:

| الضابط | المصدر | المسؤول |
|---|---|---|
| اختيار منطقة الاستضافة السعودية | `POL-400 §3.4.1` | الوكالة |
| تفعيل MFA للصلاحيات العالية | `POL-1400 §3.4.5` | الوكالة |
| ربط نفاذ لتسجيل الدخول الوطني | مستوى وطني | الوكالة |
| تحديد مدّة الاحتفاظ الرسمية | `Data Retention v1.0` | الوكالة |
| اختبار الاختراق من جهة ثالثة | `POL-2500` | الوكالة + مزوّد خارجي |
| تفعيل SIEM / SOC مركزي | `POL-1300` | الوكالة |
| توقيع اتفاقيات SLA و NDA مع المزوّدين | `POL-400 §3.3` | الوكالة |
| تدريب المستخدمين النهائيين | `POL-1500` | الوكالة |

جميع هذه النقاط ستُوثَّق في `docs/handover-security-packet.md` (المرحلة 2 من خطة التعديل).

### 2.3 الضوابط المستثناة صراحةً (قرار مُعتمد)

الضوابط التالية **خارج نطاق بصيرة** بقرار صريح من صاحب المشروع:

- **آلية حذف البيانات من قِبَل المستفيد** — الأصل أنّ هذه البيانات مُلك وزاريّ ولذوي المصلحة حقّ الاطلاع.
- **صفحة إيقاف طارئ (kill-switch) لحذف قاعدة البيانات** — غير مبرَّرة في نظام معلومات مؤسّسي.
- **إخفاء بيانات متقدم لما بعد الهوية الوطنية** — الجمهور المستهدَف (الوكالة والقيادات) له حق وصول مشروع.
- **MFA وسياسة كلمات سرّ قاسية** — مرحلة تطوير، مطوّر واحد؛ تُفعَّل من الوكالة عند النشر.

## 3. تصنيف البيانات

وفقاً لـ `DT-IS-POL-1000 V10`، البيانات في بصيرة مُصنَّفة كما يلي:

| التصنيف | أمثلة | الوصول |
|---|---|---|
| **عام (Public)** | جداول المراجع (فئات، وحدات قياس، أيام الأسبوع) | مفتوح |
| **محدود (Restricted)** | بيانات المستفيدين الشخصية، بيانات الموظفين، سجلات الرعاية اليومية، الإنذارات، الشهادات | حسب الدور |
| **سري (Confidential)** | الملف الطبي الكامل، الأدوية، البحث الاجتماعي، محاضر الحوادث | محدود لأدوار معيّنة |
| **سري للغاية (Top Secret)** | غير مُستخدَم في بصيرة | — |

التفاصيل الكاملة لكل جدول في `supabase/sql/023_data_classification.sql`.

## 6. حقوق الوصول — نموذج الأدوار

| الدور | نطاق الوصول التقني | كفاءة التفويض |
|---|---|---|
| `director` | كامل (قراءة + كتابة + إدارة) | مدير المركز فقط |
| `admin` | كامل تقنياً — مقيَّد بالوكالة | فريق النشر الوزاري |
| `doctor` | الملف الطبي + العلامات الحيوية + الأدوية (كتابة) · بيانات المستفيد الأساسية (قراءة) | طبيب المركز |
| `social_worker` | البحث الاجتماعي + الملف الاجتماعي (كتابة) · طبي (قراءة محدودة) | أخصائي اجتماعي |
| `specialist` | الملفات التخصصية (PT/Speech/Dental/Psychology) | أخصائي معتمد |
| `secretary` | الزيارات + السكرتارية + المواعيد (كتابة) · عام (قراءة) | سكرتير المركز |
| `nurse` | المتابعة اليومية + التمريض (كتابة) · طبي (قراءة ضمن المهمة) | ممرّض المركز |
| `staff` | الموظف العادي — وصول قراءة عام لمستفيديه المُعيَّنين | العاملين المباشرين |

تُطبَّق هذه الأدوار عبر:
1. `ProtectedRoute` في الواجهة (React Router v7).
2. RLS policies على مستوى Postgres (لكل جدول حساس).
3. Supabase Auth + JWT claims.
